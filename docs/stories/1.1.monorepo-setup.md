## 📖 Description

This story establishes the foundational monorepo setup that all other stories depend on. The monorepo enables:
- Shared types between frontend and backend (`packages/tipos-compartilhados`)
- Coordinated development and builds via Turborepo
- Single source of truth for dependencies
- Efficient parallel testing and linting

### Debug Log

**Challenges & Resolutions:**
1. **npm workspace reference syntax** — Initial "workspace:*" syntax not supported. Resolved by using "*" for workspace dependencies.
2. **Turbo 2.0 compatibility** — Turbo 2.0 changed "pipeline" to "tasks" in config. Updated turbo.json.
3. **Missing tsconfig.json** — TypeScript validation failed without tsconfig in each workspace. Created tsc configs.
4. **Next.js lint requirements** — `next lint` requires app/pages directory. Switched to ESLint for now, will be configured in Story 1.2.
