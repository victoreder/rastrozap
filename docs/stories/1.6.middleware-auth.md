## 📋 Description

Implement server-side middleware to protect authenticated routes and client-side auth guards to prevent unauthorized access. Establish route protection patterns that will be used across all application pages.

---

## ✅ Acceptance Criteria

- [ ] Middleware created to check authentication on protected routes
- [ ] Route protection for `/api/*` endpoints (401 if unauthenticated)
- [ ] Route redirection for `/app/*` pages to `/login` if not authenticated
- [ ] Session refresh mechanism for expired tokens
- [ ] Client-side auth context created for React components
- [ ] Protected route component `<ProtectedRoute>` created
- [ ] Auth persistence across page refreshes
- [ ] Error handling for auth failures (401, 403)
- [ ] TypeScript types for auth state and session
- [ ] Middleware tests covering main scenarios

---

## 🎯 Implementation Details

### Middleware Layer
- File: `apps/frontend/src/middleware.ts`
- Checks `auth.session` from Supabase cookies
- Redirects unauthenticated users to `/login`
- Allows public routes: `/`, `/login`, `/registro`, `/esqueci-senha`

### Auth Context
- File: `apps/frontend/src/lib/auth-context.tsx`
- Provides `useAuth()` hook
- Tracks `user`, `session`, `isLoading`, `error`
- Syncs across browser tabs using `StorageEvent`

### Protected Route Component
- File: `apps/frontend/src/components/layout/ProtectedRoute.tsx`
- Wraps pages that require authentication
- Shows loading state while checking auth
- Redirects to `/login` if unauthorized

### Session Refresh
- Auto-refresh tokens 1 minute before expiry
- Call `supabase.auth.refreshSession()` on interval
- Clear session on refresh failure
