# Story 1.8: Auth Pages (Login & Register)

**Status:** Draft
**Epic:** 1 (Foundation & Auth)
**Priority:** High
**Points:** 5
**Created:** 2026-02-21

---

## 📋 Description

Implement authentication pages (login and register) with form validation, error handling, and secure password management. These pages are the entry point for user account creation and session initiation.

---

## ✅ Acceptance Criteria

- [ ] Login page at `/login` (public)
- [ ] Register page at `/registro` (public)
- [ ] Email validation (valid email format)
- [ ] Password validation (minimum 6 characters)
- [ ] Password confirmation on register
- [ ] Error messages for auth failures (invalid credentials, account exists)
- [ ] Success handling (redirect to dashboard after login)
- [ ] Loading state during API call
- [ ] "Forgot password" link on login page
- [ ] Link to register from login and vice versa
- [ ] Responsive design (mobile-first)
- [ ] Form validation tests
- [ ] API integration tests with Supabase Auth

---

## 🎯 Implementation Details

### Login Page (`/login`)
- File: `apps/frontend/src/app/(auth)/login/page.tsx`
- Form fields: Email, Password
- Button: "Entrar" (calls `/api/auth/login`)
- Links: "Criar conta" → `/registro`, "Esqueci senha" → `/esqueci-senha`
- Error display: Toast or inline error message
- Success: Redirect to `/app/dashboard`
- Remember me: Optional (store in localStorage)

### Register Page (`/registro`)
- File: `apps/frontend/src/app/(auth)/registro/page.tsx`
- Form fields: Email, Password, Password Confirm
- Button: "Criar Conta" (calls `/api/auth/signup`)
- Links: "Já tem conta?" → `/login`
- Terms: Checkbox for terms acceptance
- Error display: Field-level + form-level errors
- Success: Redirect to `/app/dashboard` with user session

### Form Validation
- Email: Valid email format (RFC 5322)
- Password: Minimum 6 characters
- Password Confirm: Must match Password field
- Terms: Must be checked on register

### Error Handling
- Network error: Show retry message
- Invalid credentials: "Email ou senha incorretos"
- Account exists: "Email já cadastrado"
- Weak password: "Senha muito fraca (mínimo 6 caracteres)"

### Layout
- Centered card design (max-width 400px)
- Logo/branding at top
- Form with all fields visible at once
- Submit button full width
- Links to alternate pages
